🛡️ 安全最佳實踐

  • 客戶端配置：可以安全地放在前端代碼中
  • 服務帳戶私鑰：絕對不能放在前端，只用於後端服務
  • 安全規則：在 Firebase Console 中設定資料存取規則
  • API 限制：在 Google Cloud Console 中限制 API 金鑰使用範圍
  • 域名限制：只允許授權的域名存取 Firebase

🚨 緊急安全處理

如果發現安全問題，請立即執行以下操作：

立即執行：

  1. 撤銷所有可能洩露的 API 金鑰和私鑰
  2. 生成新的認證憑證
  3. 更新 Firebase 安全規則
  4. 檢查最近的存取日誌
  5. 通知相關人員
